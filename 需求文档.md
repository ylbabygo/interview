# 51talk 用户访谈会在线邀请函（移动端 H5）产品需求文档
**项目名称：** 51talk 11.29 用户访谈会邀请函 H5
**发布平台：** 移动端网页（适配微信、浏览器）
**目标用户：** 被邀请的 51talk 现有用户（家长/学员）
**活动日期：** 2025年11月29日（周六）

---

## 1. 项目背景与目标
*   **背景：** 为了优化课程与产品体验，51talk 计划举办线下用户访谈会。
*   **目标：** 通过精美的 H5 邀请函展示诚意，告知活动详情（时间、地点、酬金），并收集用户的参会意愿及具体时间段。
*   **核心利益点（Hook）：** 改善产品体验 + **800元访谈酬金**。

---

## 2. 页面流程图 (User Flow)
1.  用户打开链接 -> 进入**邀请函主页**（视觉展示+活动文案）。
2.  下滑浏览 -> 查看**活动详情**（地点、地图、酬金）。
3.  页面底部/表单区 -> 填写**称呼** + 选择**时间段**。
4.  点击**“接受邀请”** -> 系统校验输入。
5.  校验通过 -> 弹出**报名成功/详情弹窗**（再次确认信息）。

---

## 3. 前端功能需求 (Front-end)

### 3.1 页面结构与视觉风格
*   **风格调性：** 温暖、专业、高品质。建议使用 51talk 品牌色（黄/橙）搭配白色或暖灰色背景，体现“重视用户”的氛围。
*   **头图 (Header)：** 51talk Logo + “诚挚邀请”或“用户访谈会邀请函”字样。

### 3.2 内容模块
**A. 邀请寄语模块**
*   **文案内容：**
    > “想诚挚地邀请您参加我们的用户访谈活动。我们正在努力优化课程和产品，而您作为我们非常重视的用户，您的真实感受和建议对我们至关重要。我们相信，您的意见能直接帮助我们为孩子创造更好的学习体验。”
*   **重点高亮：** 将“您的意见”、“更好的学习体验”做加粗或颜色区分处理。

**B. 访谈酬金模块 (高亮展示)**
*   **设计建议：** 使用红包、礼品卡或金币图标作为视觉辅助。
*   **文案：**
    > “为了感谢您专程前来并贡献宝贵的智慧，在访谈结束后，我们会为您准备一份 **800元** 的访谈酬金，作为您交通和时间上的补贴。”

**C. 活动信息模块**
*   **活动日期：** 2025年11月29日 (周六)
*   **活动地点：**
    *   文字：北京市朝阳区东大桥路8号尚都国际中心A座2808
    *   **交互功能：** 需展示一张静态地图图片。
    *   **点击事件：** 点击地图图片或“导航”按钮，跳转至百度地图导航页面。
    *   **链接地址：** `https://j.map.baidu.com/fc/dcN`

### 3.3 数据采集表单 (交互核心)
在页面下方设置报名表单：
1.  **称呼 (Input)：**
    *   类型：文本输入框
    *   提示语(Placeholder)：请输入您的称呼（如：王女士/李先生）
    *   校验：必填，不能为空。
2.  **参会时间段 (Radio Group)：**
    *   类型：单选按钮（二选一）
    *   选项 A：11月29日 上午 9:30-11:30
    *   选项 B：11月29日 下午 14:00-16:00
    *   校验：必选。
3.  **提交按钮 (CTA)：**
    *   文案：“接受邀请”
    *   状态：点击后呈加载状态，防止重复提交。

### 3.4 提交反馈 (Pop-up)
用户点击“接受邀请”并提交成功后，**弹出**详情确认弹窗：
*   **标题：** 报名成功！期待您的到来
*   **内容：**
    *   您的称呼：[用户输入的称呼]
    *   您的场次：[用户选择的时间段]
    *   *温馨提示：请您截图保存此页面，届时凭此信息签到。*
*   **按钮：** “我知道了”或“关闭”

---

## 4. 后台功能需求 (Back-end)

需要开发一个简易的后台管理页面（或直接导出数据的功能），用于市场/运营人员统计名单。

### 4.1 数据结构 (Data Schema)
数据库需存储以下字段：

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | Int | 自增ID |
| `user_name` | String | 用户填写的称呼 |
| `selected_slot` | String | 选项值：'AM'(上午) 或 'PM'(下午) |
| `submit_time` | Timestamp | 用户提交的时间 |
| `ip_address` | String | 用户IP (可选，防刷) |

### 4.2 后台管理列表
运营人员登录后台后，需要看到一个列表，包含：
1.  **序号**
2.  **用户称呼**
3.  **选择的时间段**（上午 9:30-11:30 / 下午 14:00-16:00）
4.  **提交时间**

### 4.3 导出功能
*   提供 **“导出 Excel”** 按钮。
*   导出格式：`.xlsx` 或 `.csv`。
*   用途：方便工作人员打印名单进行现场签到核对。

---

## 5. 非功能性需求 (Non-functional)
1.  **兼容性：** 必须完美适配主流手机屏幕（iOS/Android），重点适配微信内置浏览器。
2.  **加载速度：** 图片需压缩，保证在4G/5G网络下秒开。
3.  **分享设置：** 当用户将链接发送给朋友或朋友圈时，自定义分享卡片：
    *   标题：51talk 用户访谈会诚挚邀请
    *   描述：期待您的参与，共创更好产品体验。
    *   图标：51talk Logo。

---

## 6. 开发与交付建议
*   **第一阶段（UI设计）：** 输出高保真设计图（1天）。
*   **第二阶段（开发）：** 前端页面切图+后端接口对接（1-2天）。
*   **第三阶段（测试）：** 内部测试表单提交、地图跳转、数据导出是否正常（0.5天）。

---

**给开发者的附注：**
*   地图导航链接请务必测试在微信环境下的跳转是否能够唤起百度地图小程序或网页版。
*   表单提交建议增加简单的防抖动（Debounce）处理，避免用户多次点击导致后台数据重复。
